Get Started

This is a short guide to help you get going quickly with a typical configuration. For in-depth information about available features and functionality, consult the user manual.

Requirements

The only hard requirement is Java 7+. Particular resolvers and processors may have additional requirements.

Download

Download

Configuration

The distribution archive bundles a sample configuration file, named cantaloupe.properties.sample. Do not edit this file—instead, create a copy and edit that.

To get started quickly, just set FilesystemResolver.BasicLookupStrategy.path_prefix to a folder containing some images, and leave all the other options alone.

Running

From the command prompt:

$ java -Dcantaloupe.config=/path/to/cantaloupe.properties -Xmx2g -jar Cantaloupe-x.x.x.jar

(-Xmx2g tells the JVM to use up to 2GB of heap space [RAM, sort of]. Without this argument, it may default to a much smaller amount, leading to OutOfMemoryErrors.)

Cantaloupe is now running at: http://localhost:8182/. To test, assuming you have an image named image.tif, try accessing:

  • http://localhost:8182/iiif/2/image.tif/info.json
  • http://localhost:8182/iiif/2/image.tif/0,0,200,200/full/0/default.jpg

Upgrading

Since instances are self-contained, new versions can run alongside existing ones, with each using its own configuration file. There may be changes to the file structure from version to version, though, so check below to see if there is anything more to be done.

2.1 to 2.2

  • Add the watermark.* keys from the sample configuration.
  • Add the PdfBoxProcessor.* keys from the sample configuration.
  • Add the cache.server.worker.* keys from the sample configuration.

2.0 to 2.1

  • Add the JaiProcessor.* keys from the sample configuration.
  • Add the Java2dProcessor.tif.compression key from the sample configuration.
  • Add the log.*.SyslogAppender.* keys from the sample configuration.
  • Add the FilesystemCache.dir.* keys from the sample configuration.
  • Add the authorized? method from the sample delegate script.
  • Purge your cache.

1.2 to 2.0

There are so many new and renamed configuration keys in this version that it is recommended to start over with a new configuration file. The inline comments, which serve as the primary source of configuration documentation, have also been updated.

Also, because the cache naming format has changed, it will be necessary to purge your cache before using this version.

Note that the Image API 2.0 endpoint, which used to be located at /iiif, has moved to /iiif/2.

Also, the landing page has moved to /.

1.2 to 1.2.1

Nothing to do.

1.1 to 1.2

  • Add the http.content_disposition key from the sample configuration.
  • Add the AmazonS3Resolver.* keys from the sample configuration.
  • Add the FfmpegProcessor.path_to_binaries key from the sample configuration.
  • Add the cache.server.resolve_first key from the sample configuration.
  • Remove the generate_https_links key from your configuration file, and add the base_uri key from the sample configuration.

1.0.x to 1.1

  • Add the cache.client.enabled key from the sample configuration.
