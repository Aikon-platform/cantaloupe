What's New in 3.2

3.2

  • All processors that use the Java 2D pipeline (Java2dProcessor, KakaduProcessor, OpenJpegProcessor, PdfBoxProcessor, FfmpegProcessor) use a custom box filter for downscaling and a bicubic filter for upscaling, resulting in much better quality and 100-400% faster scaling performance.
  • Added the ability to embed ICC color profiles in derivative images.
  • Increased the default Java2dProcessor/JaiProcessor JPEG output quality to 0.8 (80).
  • When an output format is requested that is valid but unsupported by the current processor, HTTP 415 is returned instead of HTTP 400.
