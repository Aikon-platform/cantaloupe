Overview

Many image formats are capable of storing metadata alongside the pixel data in an image file. Metadata is diverse and may include anything from information about the capture device (camera, scanner, etc.) to keywords to copyright information.

When Cantaloupe receives a request for an image—unless the request is for the full-sized unmodified source image—it will have to create and return a derivative image based on this source image. The derivative image is a whole new image, distinct from the source image, and may not even be in the same format as the source image. The metadata contained in the source image would ordinarily not be copied over.

When metadata.preserve is set to true in the configuration file, Cantaloupe will make an effort to copy any source metadata it finds into derivative images. This is not foolproof—some formats don't support all types of metadata, and metadata may be encoded differently in one format than another. Cantaloupe is generally aware of EXIF, IPTC (IIM), and XMP metadata.

Processor Support

Most processors support metadata preservation to at least some extent.

Implications

Performance

As of this writing, the performance penalty of the metadata preservation feature hasn't been measured. Though it is likely to be small, this feature is disabled by default.

Caching

Metadata settings are encoded in cached-image identifiers, so derivative images with and without metadata are cached separately.
