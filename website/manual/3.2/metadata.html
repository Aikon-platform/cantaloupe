Metadata

  • Overview
  • Implications
    • Performance

Overview

Many image formats are capable of supplementing image data in a file with metadata describing it. In addition to format-specific options, various standards exist to encode different types of metadata in a consistent way across file types. More than one such type may be present even within the same file.

When Cantaloupe receives a request for an image—unless the request is for the full-sized unmodified source image—it will have to create and return a derivative image based on a source image. Because the derivative is a whole new image, distinct from the source image (and maybe not even in the same format), the metadata would not be automatically copied over.

When metadata.preserve is set to true in the configuration file, Cantaloupe will make an effort to copy source metadata into derivative images.

Most processors are aware of EXIF, IPTC (IIM), and XMP metadata in most formats that support it. Currently, the metadata-preservation feature generally does not work across formats.

Implications

Performance

Metadata preservation tends to incur a performance penalty, and it is disabled by default.

Caching

Metadata settings are encoded in cached-image identifiers, so derivative images with and without metadata are cached separately.
