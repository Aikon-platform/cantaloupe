Methods

Cantaloupe can obtain its configuration from a file or from operating system environment variables.

Properties File

Properties file configuration is the traditional configuration method that Cantaloupe has used since its inception. (.properties is a syntax for configuration files that is very easy to read and write.) It is still the recommended configuration method.

A sample .properties configuration file, called cantaloupe.properties.sample, is included in the distribution archive. It is recommended to make a copy of this file and use the copy as your primary configuration file.

To use a .properties configuration file, supply the -Dcantaloupe.config VM option, pointing to an absolute or relative path to the file. The file can be located anywhere, but its name must end in .properties. See Getting Started for examples.

Environment

Environment configuration uses configuration stored in the user environment. This method debuted in version 3.3.

Note that because the syntax of the keys used in other configuration methods is incompatible with the environment, they need to be modified to be all-uppercase; have a prefix of CANTALOUPE_; and have underscores in place of periods. So, a key like:

endpoint.iiif.2.enabled = true

Would be stored in the environment as:

CANTALOUPE_ENDPOINT_IIIF_2_ENABLED=true

Cantaloupe cannot write to the environment, but it will pick up changes immediately.

Switching to a Different Method

There are two main options for switching configuration methods. The first is two transform your current configuration to your desired configuration manually. The second is to use the "Configuration" tab of the Control Panel to retrieve a different representation your current configuration.

Live Changes

For file-based configuration methods, Cantaloupe will automatically reload the configuration file within a few seconds of a change. Changes to most keys take effect immediately, but some require a restart; see the sample properties file for documentation of which.

Remote Changes

The configuration can be updated remotely via the REST API. When a change is received, it will be activated immediately and written out to the configuration file, if available (but not the environment, as that is read-only).
