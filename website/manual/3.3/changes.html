What's New in 3.3

  • Added textual watermarking.
  • Added RESTful API endpoints to purge images from the cache and retrieve and update the configuration.
  • Added the ability to redirect to another URL via the authorized?() delegate method.
  • Added an auto-normalize option to all image processors.
  • ImageMagickProcessor supports ImageMagick 7.
  • FfmpegProcessor uses a Java 2D processing pipeline rather than doing all processing with ffmpeg.
  • Auto-computed tile sizes of source images with an odd number of pixels in a dimension in information responses are rounded up, to prevent clients from requesting narrow edge tiles.
  • Added an optional delegate method cache.
  • A cache=false URL query option can be used in all endpoints to bypass the derivative cache per-request.
  • Operations that would have no effect in their context—for example, a crop to 300x200, followed by a scale-to-fit to 300 pixels wide—are skipped.
  • The delegate script is reloaded on change rather than per-request.
  • Added support for PDFs containing JBIG2-encoded images to PdfBoxProcessor.
  • In standalone mode, the application is initialized at startup rather than upon the first request.
