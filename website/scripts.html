Custom Scripts

Several aspects of Cantaloupe offer the ability to use custom Ruby functions as "hooks" to provide dynamic information back to the image server. While the exact details of each script vary, all scripts do have to obey some general rules.

Naming

Scripts can be named anything, but they must have a .rb extension. Then, they can be referenced in a configuration option by either their full absolute pathname, or their filename:

# Option 1: absolute pathname
some_key = /path/to/functions.rb

# Option 2: filename only; will be expected to be located in the
# same folder as the config file, or the current working directory
some_key = functions.rb

Example

A script file must contain a function, but the name, parameters, and return value will vary. Here is an example of a script used by FilesystemResolver that performs a Solr query to return a pathname based on an identifier:

# functions.rb
require 'net/http'

def get_pathname(identifier)
  uri = 'http://localhost:8983/solr/collection1/select?q=' +
      URI.escape('id:"' + identifier + '"') +
      '&fl=pathname_si&wt=ruby'
  uri = URI.parse(uri)

  http = Net::HTTP.new(uri.host, uri.port)
  request = Net::HTTP::Get.new(uri.request_uri)
  response = http.request(request)
  return nil if response.code.to_i >= 400

  results = eval(response.body)['response']['docs']
  results.any? ? results.first['pathname_si'] : nil
end

# Uncomment to test on the command line (`ruby functions.rb`)
# puts get_pathname('identifier-to-test')

Inside a function, anything goes — it will be interpreted by the JRuby interpreter built into Cantaloupe, so you can even use any (non-platform-native) gems that you have installed with gem install. Because it will be called upon each image request, though, it should be written with efficiency in mind.

Note: typically, neither function arguments nor return values are sanitized or validated. Be very careful to write defensive, injection-aware, well-tested code.
Note: the first time a script runs, it will take a long time to execute. That is just JRuby loading. Subsequent runs will be much faster.
Note: scripts will be re-read on each request, so they can be edited without restarting.
